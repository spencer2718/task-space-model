## Executive Summary

This patch implements external validation of the GWA-based activity geometry. The goal is to determine whether occupation-pair overlap (computed from our kernel matrix) predicts real economic frictions (worker mobility or wage comovement). This is a **rejection diagnostic**—we're testing whether to proceed with Phase II or pivot to a different geometry.

**Success criterion:** β > 0 with p < 0.10 in the validation regression
**Failure response:** Document null result, recommend DWA domain or Recipe Y pivot

| Percentile | Description |
|------------|-------------|
| p10 | Narrow kernel (minimal spillover) |
| p25 | Below median |

- If all five show β > 0 with p < 0.10: Strong pass
- If only some pass: Note which σ range works; consider implications
- If none pass: Clear fail; geometry doesn't work at any spillover scale

```
  - Relaxed "diagonal is max in row" test: kernel-weighted overlap doesn't guarantee
    self-overlap is highest; two occupations can have higher overlap than self if
    their distributions center on nearby activities.
- Observations:
  - 894 occupations, 399,171 pairs
  - All 5 sigma values produce similar mean overlaps (~0.025)
  - Overlap variance decreases with sigma (wider kernel = more diffuse)
```

## Anti-Patterns to Avoid

1. **Cherry-picking σ:** Don't search for the σ that gives best p-value
2. **Dropping inconvenient pairs:** Document all exclusions with rationale
3. **Over-controlling:** Baseline spec has no controls; controls are robustness only
4. **Selective reporting:** Report all specifications run, not just favorable ones
5. **Premature optimization:** If GWA fails, that's useful information—don't rescue it
